## 1. 프로젝트 개요

### 컨셉
- 전국 방탈출 매장과 테마 정보를 **지도와 리스트**로 한눈에 보여줌
- 사용자 리뷰와 별점으로 방탈출 선택을 돕는 모바일 앱
- **차별화 포인트**: 지도 색칠로 전국 방탈출 현황 시각화 + 도장깨기 게이미피케이션

### 타겟 사용자
- 방탈출을 자주 즐기는 20~30대
- 처음 방탈출을 시도해보려는 입문자
- 방탈출 마니아 (정복 욕구 자극)

### 개발자 배경
- Java/Vue.js 2년차 개발자
- 모바일 앱 개발 경험 없음 → 이번 프로젝트로 학습
- 사이드 프로젝트로 시작 → 실제 출시 + SRE 학습 동시 목표

### 목표
1. **단기 (3~4달)**: MVP 출시 (지도+리스트+검색+리뷰+지도색칠)
2. **장기**: 백엔드 운영 경험을 SRE/DevOps 학습으로 연결 (Docker, K8s, AWS, 모니터링)

## 2. 핵심 기능 정의

### MVP (1차 출시)
1. ✅ 매장/테마 리스트 보기
2. ✅ 매장 상세 + 테마 리스트
3. 🔲 테마 상세 + 리뷰 리스트
4. ⭐ **지도 색칠 (매장 수 모드)** — 핵심 차별화
5. 🔲 검색/필터 (지역, 장르, 난이도)
6. 🔲 카카오 로그인
7. 🔲 리뷰/별점 작성 (10개 이상 시 평점 공개)
8. 🔲 외부 예약 링크 연결

### Phase 2 확장
9. ⭐ **지도 색칠 모드 추가**: 평균 평점 모드 + 도장깨기 모드
10. 즐겨찾기
11. 내가 푼 방 기록
12. 푸시 알림

### Phase 3 고도화
13. 사진/태그/좋아요
14. 신고 기능
15. 랭킹/배지
16. AI 리뷰 요약

### 제외 (안 함)
- ❌ 자체 예약 시스템 (외부 링크 연결로 대체)
- ❌ 크롤링 (법적 리스크, 직접 입력으로 시작)

## 3. 화면 구성

### MVP 화면 (10개)

```
1. 홈/지도 화면 (메인)
   ├── 모드 1: 📊 매장 수 색칠
   ├── 모드 2: 🌟 평균 평점 색칠 (Phase 2)
   └── 모드 3: 🏆 내 도장 색칠 (Phase 2)

2. 매장 리스트 화면
   └── 검색/필터 가능

3. 검색/필터 화면
   ├── 키워드 검색
   ├── 지역 필터 (홍대/강남/합정...)
   ├── 장르 필터 (추리/공포/SF/판타지...)
   └── 난이도 필터 (1~5)

4. 매장 상세 화면
   ├── 매장 정보 (이름, 주소, 영업시간)
   ├── 테마 리스트
   └── 외부 예약 링크 버튼

5. 테마 상세 화면
   ├── 테마 정보 (장르, 난이도, 시간, 인원, 가격)
   ├── 평점 ⭐ (10개 이상 시) 또는 "리뷰 모으는 중"
   ├── 리뷰 리스트 (3개 미리 + 전체 보기)
   └── 우하단 FAB: "리뷰 쓰기"

6. 전체 리뷰 화면
   └── 정렬: 최신 / 별점 / 도움 많은

7. 리뷰 작성 화면
   ├── 별점 입력 (★★★★★)
   ├── 텍스트 입력 (30자 이상)
   ├── 성공/실패
   ├── 클리어 시간, 인원, 힌트 (선택)
   └── 방문 날짜

8. 로그인 화면
   └── 카카오 로그인 버튼만

9. 마이페이지
   ├── 내 정보
   ├── 내가 쓴 리뷰
   ├── 내가 푼 방 기록
   └── 즐겨찾기

10. 지도 상세 (지역 탭 시)
    ├── 지역 정보 (등급, 매장 수)
    ├── 해당 지역 매장 카드 5개 미리
    └── "전체 보기" → 매장 리스트
```

### 화면 흐름

```
[홈/지도] → [매장 리스트] → [매장 상세] → [테마 상세] → [리뷰 작성]
                                                    └─→ [예약 외부 링크]
                                                    └─→ [전체 리뷰]
[홈/지도] → 지역 탭 → [지역 상세] → [매장 리스트]
[홈/지도] → [검색/필터] → 결과 → [매장 상세]
[로그인] → [마이페이지]
```

## 4. 데이터 모델

### Store (매장)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | Long | PK |
| name | String | 매장명 |
| region | String | 권역 (서울/경기/충청/경상/전라/강원/제주) |
| subRegion | String | 세부 지역 (홍대/강남/합정...) |
| address | String | 도로명 주소 |
| latitude | Double | 위도 |
| longitude | Double | 경도 |
| phone | String | 연락처 |
| businessHours | String | 영업시간 |
| thumbnailUrl | String | 대표 이미지 URL |
| themeCount | Integer | 테마 개수 (계산) |
| averageRating | Double | 평균 별점 (계산, 10개+ 일 때만) |
| reviewCount | Integer | 전체 리뷰 수 (계산) |
| createdAt | DateTime | 등록일 |

### Theme (테마)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | Long | PK |
| storeId | Long | FK → Store |
| name | String | 테마명 |
| genre | String | 장르 (추리/공포/SF/판타지/감성/스릴러) |
| difficulty | Integer | 난이도 1~5 |
| minPeople | Integer | 최소 인원 |
| maxPeople | Integer | 최대 인원 |
| durationMin | Integer | 소요 시간 (분) |
| price | Integer | 1인당 가격 |
| description | String | 한 줄 설명 |
| thumbnailUrl | String | 테마 이미지 |
| reservationUrl | String | 외부 예약 페이지 URL |
| averageRating | Double | 평균 별점 (10개+) |
| reviewCount | Integer | 리뷰 개수 |
| ratingVisible | Boolean | 평점 공개 여부 (reviewCount >= 10) |

### User (사용자)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | Long | PK |
| kakaoId | String | 카카오 OAuth ID (Unique) |
| nickname | String | 닉네임 |
| profileImage | String | 프로필 이미지 URL |
| createdAt | DateTime | 가입일 |

### Review (리뷰)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | Long | PK |
| themeId | Long | FK → Theme |
| userId | Long | FK → User |
| rating | Double | 별점 1.0~5.0 (0.5 단위) |
| content | String | 텍스트 후기 (30자 이상) |
| isSuccess | Boolean | 탈출 성공 여부 |
| clearTimeMin | Integer | 클리어 시간 (분, 선택) |
| hintsUsed | Integer | 사용한 힌트 수 (선택) |
| peopleCount | Integer | 함께한 인원 (선택) |
| playDate | Date | 방문 날짜 |
| createdAt | DateTime | 작성일 |
| editCount | Integer | 수정 횟수 (최대 1) |

**제약사항**:
- 같은 사용자가 같은 테마에 1개의 리뷰만 작성 가능 (UNIQUE: themeId + userId)
- 가입 후 24시간 이내 작성 차단 (어뷰징 방지)

### ClearLog (클리어 기록 - Phase 2 도장깨기용)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | Long | PK |
| userId | Long | FK → User |
| themeId | Long | FK → Theme |
| storeId | Long | FK → Store (조회 편의용 중복) |
| region | String | 권역 (조회 편의용) |
| clearedAt | DateTime | 클리어 시점 (Review 작성 시 자동 생성) |

→ Review 작성 시 자동으로 생성. 도장깨기 색칠의 기준 데이터.

## 5. API 엔드포인트

## 6. 지도 색칠 기능 상세 설계

### 권역 구분 (7개)

| 권역 | 포함 행정구역 |
|------|--------------|
| **서울** | 서울특별시 |
| **경기/인천** | 경기도 + 인천광역시 |
| **강원** | 강원특별자치도 |
| **충청** | 충북 + 충남 + 대전 + 세종 |
| **경상** | 경북 + 경남 + 부산 + 대구 + 울산 |
| **전라** | 전북 + 전남 + 광주 |
| **제주** | 제주특별자치도 |

### 3개 모드 (토글)

#### 모드 1: 📊 매장 수 모드 (Phase 1)

| 등급 | 매장 수 | 색상 | HEX |
|------|---------|------|-----|
| **S급** | 50개 이상 | 🟥 진한 빨강 | `#C62828` |
| **A급** | 30~49개 | 🟧 주황 | `#EF6C00` |
| **B급** | 15~29개 | 🟨 노랑 | `#F9A825` |
| **C급** | 5~14개 | 🟩 연두 | `#9E9D24` |
| **D급** | 1~4개 | ⬜ 회색 | `#BDBDBD` |
| **없음** | 0개 | ⬛ 짙은 회색 | `#757575` |

#### 모드 2: 🌟 평균 평점 모드 (Phase 2)

지역별 매장 평점의 평균. 짙은 초록 = 평점 높음, 노랑 = 낮음.

#### 모드 3: 🏆 도장깨기 모드 (Phase 2)

사용자가 그 지역에서 클리어한 매장 비율.

| 진행도 | 색상 |
|--------|------|
| 0% | 회색 (안 가봄) |
| 1~20% | 연한 파랑 |
| 21~50% | 중간 파랑 |
| 51~80% | 진한 파랑 |
| 81~99% | 보라 |
| 100% 정복 | 🌈 무지개 (특별 효과) |

### UX 흐름

```
[지도 화면 진입]
   ↓
[전국 지도 색칠 표시] (기본: 매장 수 모드)
   ↓
사용자가 지역 탭 (예: 경기/인천)
   ↓
[하단 시트 슬라이드 업]
   - 경기/인천: A급 (35개 매장, 평균 ⭐4.3)
   - 매장 미리보기 5개
   - "전체 보기" → 매장 리스트
```

### 기술 구현

**Phase 1: SVG 한국 지도**
- 무료 한국 행정구역 SVG 사용 (`naturalearthdata.com` 등에서 라이센스 확인)
- 각 `<path>` 의 `fill` 속성을 색상에 맞게 변경
- 클릭 이벤트로 권역 선택

**Phase 2 이상: flutter_map + GeoJSON**
- 진짜 지도 위에 폴리곤 오버레이
- 줌/팬 자유로움

## 7. 평점/리뷰 시스템 상세 설계

### 평점 정책

- **대상**: 테마별 (매장은 테마들의 평균값을 자동 계산)
- **척도**: 5점 만점, 0.5점 단위 (★★★★½)
- **공개 시점**: **리뷰 10개 이상 누적 시**부터 평점 표시
- **10개 미만**: "리뷰 모으는 중 (3/10)" 표시

### 리뷰 작성 정책

**필수:**
- ⭐ 별점 (1.0~5.0, 0.5 단위)
- 텍스트 후기 (**30자 이상**)
- 성공/실패 (탈출 여부)

**선택 (Phase 1):**
- 클리어 시간 (분)
- 사용한 힌트 수
- 함께한 인원
- 방문 날짜

**선택 (Phase 2):**
- 사진 (3장까지)
- 태그 (#재미있음 #스토리굿)

### 어뷰징 방지

**Phase 1 (기본):**
- ✅ 카카오 로그인 사용자만 작성
- ✅ 한 테마당 1개 리뷰만 작성 가능 (UNIQUE 제약)
- ✅ 수정은 1번까지 가능
- ✅ 삭제 후 재작성 불가
- ✅ 신고 버튼 (운영자 이메일 알림)
- ✅ 3회 이상 신고 누적 시 자동 숨김

**Phase 2 (체계화):**
- 가입 후 24시간 이내 작성 차단
- 동일 IP 빠른 다중 작성 차단
- 욕설 사전 기반 자동 필터
- 사용자 차단 기능

### 평점 표시 방식

**테마 카드:**
```
┌────────────────────────────┐
│ 추리의 끝          ⭐ 4.3   │
│ 추리 · 60분 · 28,000원      │
│ (리뷰 47개)                 │
└────────────────────────────┘

또는
```

## 12. 트러블슈팅 메모

작업 중 겪은 주요 이슈와 해결법.

### 1. 카카오맵 도메인 인증 401
- 원인: 등록 도메인에 포트 누락
- 해결: `localhost:5500`, `127.0.0.1:5500` 둘 다 등록

### 2. 카카오맵 ERR_BLOCKED_BY_CLIENT
- 원인: TouchEn PC보안 등 한국 보안 프로그램이 `dapi.kakao.com` 차단
- 해결: Edge 사용 + 추적 방지 OFF

### 3. 카카오맵 NotAuthorizedError
- 원인: 카카오 개발자 콘솔에서 카카오맵 서비스 사용 설정 OFF
- 해결: 제품 설정 → 카카오맵 → 사용 설정 ON

### 4. Spring Boot 포트 충돌 (8080, 8081, 2988 모두 사용 중)
- 원인: 다른 백그라운드 서비스들이 점유
- 해결: `application.properties`에 `server.port=3000` 명시
- 추가: `taskkill /F /IM java.exe` 으로 좀비 자바 정리

### 5. Flutter `pub add http` git ownership 에러
- 원인: Flutter SDK 폴더 소유자 불일치
- 해결: `git config --global --add safe.directory C:/src/flutter`

### 6. Flutter Chrome 자동 실행 실패
- 원인: `-d chrome` 옵션이 Chrome 디버그 모드 실행 실패
- 해결: `flutter run -d web-server` 사용 → 수동으로 URL 열기

### 7. cmd 창 갑자기 닫힘
- 원인: cmd가 첫 실행 에러 처리 못 함
- 해결: PowerShell 사용

### 8. Korean text 깨짐 in Flutter
- 원인: HTTP 응답을 그냥 `json.decode(response.body)`로 파싱
- 해결: `json.decode(utf8.decode(response.bodyBytes))` 사용

## 13. Claude Code 참고사항

다음 사항들 미리 알고 있으면 좋아요.

### 코드 작업 시 주의

1. **Spring Boot DevTools가 깔려 있어** → 코드 수정하면 자동 재시작. 따로 서버 재시작 명령 불필요.

2. **CORS 이미 열림** → `WebConfig.java`에서 `/api/**` 모든 출처 허용 중. 새 엔드포인트 추가해도 Flutter에서 바로 호출 가능.

3. **Flutter Hot Reload** → `r` 키. 안 먹히면 `R` 키 (Hot Restart). 그래도 안 되면 재시작.

4. **utf8 디코딩 필수** → 한글 깨짐 방지. `json.decode(utf8.decode(response.bodyBytes))` 패턴 사용.

5. **데이터는 전부 하드코딩** → DB 도입 전이라 서버 재시작하면 상태 초기화. 진짜 영구 저장은 Phase 2부터.

6. **포트는 3000 고정** → 변경 시 Flutter의 API URL도 같이 수정 필요.

7. **`Theme` 이름 충돌 주의** → Flutter에 내장된 `Theme` 클래스가 있음. 모델은 `EscapeTheme` 이름 사용 중.

### 다음 작업 우선순위

본 기획에 따라 다음 순서로 진행 권장:
