# 방방 (가칭) — 방탈출 지도 앱 프로젝트

> 전국 방탈출 매장과 테마를 한눈에 보여주는 모바일 앱
> SRE/DevOps 학습을 겸한 실전 풀스택 프로젝트

---

## 프로젝트 개요

### 컨셉
- 전국 방탈출 매장과 테마 정보를 **지도와 리스트**로 한눈에 보여줌
- 사용자 리뷰와 별점으로 방탈출 선택을 돕는 모바일 앱
- 처음에는 직접 입력한 30~50개 매장으로 시작, 점차 확장

### 개발자 배경
- Java/Vue.js 2년차 개발자
- 모바일 앱 개발 경험 없음
- Spring Boot는 회사에서 사용 경험 있음
- 사이드 프로젝트로 시작 → 실제 출시 + SRE 학습 동시 목표

### 목표
1. **단기**: MVP 출시 (지도+리스트+검색+리뷰)
2. **장기**: 백엔드 운영 경험을 SRE/DevOps 학습으로 연결 (Docker, K8s, AWS, 모니터링)

## 현재 구현 상태

### ✅ 완성된 것

1. **카카오맵 웹 페이지** (`web/map_test.html`)
   - 카카오맵 JavaScript SDK 사용
   - 홍대 인근 매장 5개를 마커로 표시
   - 커스텀 오버레이로 매장명 라벨 표시
   - 마커 클릭 시 alert

2. **Spring Boot REST API** (`backend/`)
   - **포트 3000** (8080은 시스템에서 사용 중이라 변경)
   - **CORS 허용** (`/api/**` 모든 출처)
   - 엔드포인트:
     - `GET /api/stores` - 매장 전체 목록 (5~6개)
     - `GET /api/stores/{id}` - 매장 하나 상세
   - 데이터는 `StoreController.java` 안에 하드코딩

3. **Flutter 모바일 앱** (`mobile/`)
   - **Web Server 모드**로 실행 (`flutter run -d web-server`)
   - Spring Boot API에서 매장 데이터 받아와 표시
   - 화면 구성:
     - 상단 앱바 ("방방" + 새로고침/검색 아이콘)
     - 매장 카드 리스트 (번호, 이름, 주소, 태그)
     - 하단 네비게이션 바 (지도/리스트/즐겨찾기/마이 — 시각만 있음)
   - 상태 처리:
     - 로딩 중 → 스피너
     - 에러 시 → 빨간 에러 화면 + "다시 시도" 버튼
     - 정상 시 → 매장 리스트 + "🟢 API 연결됨" 뱃지
   - Pull-to-refresh 동작

### Flutter — Store 모델

```dart
class Store {
  final int id;
  final String name;
  final String region;
  final String address;
  final String genre;
  final int themeCount;
  final double latitude;
  final double longitude;

  Store({
    required this.id,
    required this.name,
    required this.region,
    required this.address,
    required this.genre,
    required this.themeCount,
    required this.latitude,
    required this.longitude,
  });

  factory Store.fromJson(Map<String, dynamic> json) {
    return Store(
      id: json['id'],
      name: json['name'],
      region: json['region'],
      address: json['address'],
      genre: json['genre'],
      themeCount: json['themeCount'],
      latitude: (json['latitude'] as num).toDouble(),
      longitude: (json['longitude'] as num).toDouble(),
    );
  }
}
```

```dart
Future<List<Store>> fetchStores() async {
  final url = Uri.parse('http://localhost:3000/api/stores');
  final response = await http.get(url);

  if (response.statusCode == 200) {
    final List<dynamic> data = json.decode(utf8.decode(response.bodyBytes));
    return data.map((item) => Store.fromJson(item)).toList();
  } else {
    throw Exception('매장 정보를 불러오지 못했습니다 (${response.statusCode})');
  }
}
```

## 트러블슈팅 메모

작업 중 겪었던 이슈와 해결법.

### 1. 카카오맵 도메인 인증 401 에러
- **원인**: 카카오 개발자 콘솔에 `http://localhost`만 등록했는데, Live Server는 `http://127.0.0.1:5500`을 사용 → 포트 차이로 401
- **해결**: `http://localhost:5500`, `http://127.0.0.1:5500` 둘 다 등록

### 2. 카카오맵 ERR_BLOCKED_BY_CLIENT
- **원인**: TouchEn PC보안 등 한국 보안 프로그램이 `dapi.kakao.com` 호출 차단
- **해결**: Edge 브라우저 사용 + Edge 추적 방지 OFF

### 3. 카카오맵 NotAuthorizedError "OPEN_MAP_AND_LOCAL service disabled"
- **원인**: 카카오 개발자 콘솔에서 "카카오맵" 제품 사용 설정 OFF 상태
- **해결**: 카카오 개발자 콘솔 → 제품 설정 → 카카오맵 → 사용 설정 ON

### 4. Spring Boot 포트 8080, 8081, 2988 등 사용 중
- **원인**: 본인 PC에 다른 백그라운드 서버들이 잡고 있음
- **해결**: `application.properties`에 `server.port=3000` 명시. (3000은 비어 있었음)
- **참고 명령**: 
  - 안 쓰는 포트 찾기: PowerShell 한 줄 명령
  - `netstat -ano | findstr :3000` 으로 점유 확인
  - `taskkill /F /IM java.exe` 으로 좀비 자바 프로세스 정리

### 5. Flutter `pub add http` 시 git ownership 에러
- **원인**: Flutter SDK 폴더의 소유자가 Administrators인데 일반 사용자로 접근 → Git이 보안상 차단
- **해결**: `git config --global --add safe.directory C:/src/flutter`

### 6. Flutter Chrome 자동 실행 실패
- **원인**: `flutter run -d chrome`이 Chrome 디버그 모드 실행에 실패
- **해결**: `flutter run -d web-server` 사용 → 수동으로 URL 열기

### 7. cmd 창이 갑자기 닫힘 (`flutter --version`)
- **원인**: cmd가 첫 실행 에러를 처리 못 함
- **해결**: PowerShell 사용 (창이 안 닫히고 에러 메시지 보임)

## Claude Code로 이어갈 때 참고할 것

1. **Spring Boot DevTools가 깔려 있어서 코드 수정하면 자동 재시작.** 따로 서버 재시작 명령 안 해도 됨.
2. **CORS 이미 열려있음.** 새 엔드포인트 추가해도 Flutter에서 바로 호출 가능.
3. **Flutter는 hot reload 가능** (`r` 키), hot restart는 `R` 키. 그래도 가끔 안 먹히면 재시작.
4. **utf8 디코딩 필수** — `json.decode(utf8.decode(response.bodyBytes))` 안 하면 한글 깨짐.
5. **데이터는 전부 하드코딩 상태** — DB 도입 전이라 서버 재시작하면 모든 상태 초기화.
6. **포트는 3000 고정**. 변경 시 Flutter의 `fetchStores()` URL도 같이 수정 필요.
7. **`Theme`은 Flutter의 예약어**일 수도 있으니, 모델 클래스명을 `EscapeTheme` 등으로 바꾸는 게 안전.
